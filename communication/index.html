    // subsys_channels

    Four ways to reach anyone,
    none of them leaky.

    • sec·01

      Text

      Sealed-sender messages. The relay routes the envelope and never sees the letter.

      aes-256-gcm · x25519
    • sec·02

      Voice

      Sub-40 ms hops with jitter smoothing that doesn't phone home to tune itself.

      opus · srtp
    • sec·03

      Presence

      Say who's online without telling the server who's asking.

      private set intersection
    • sec·04

      Transfer

      Files stream peer to peer. The relay is a fallback, not a filing cabinet.

      webrtc · fallback tls

    // subsys_protocol

    The handshake, in full.

    1. 00 Keypair

      Generated on device. The private half never serialises.

    2. 01 Handshake

      Ephemeral X25519. Forward secrecy per session, not per account.

    3. 02 Route

      Envelope hops the relay. Sender identity is stripped at ingress.

    4. 03 Seal

      Delivered, decrypted on device, receipt is optional and off by default.

    [note] The relay is deliberately dumb. It knows an envelope arrived and where to push it next. It does not know who sent it, what it says, or that you and the recipient have ever spoken before.
